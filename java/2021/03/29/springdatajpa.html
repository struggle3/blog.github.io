hibernate中的javax.persistence介绍

2021-03-29

JPA(Java Persistence API)是Sun官方提出的Java持久化规范，它为Java开发人员提供了一种对象关系映射的方式管理Java应用中的关系数据。

它显示了如何定义一个面向普通Java对象（POJO）作为一个实体，以及如何与管理关系实体提供一套标准。因此javax.persistence下面的有些注解还是必须要去了解的，以便于更好地提高工作效率。

Hibernate是它的一种实现方式，还有EclipseLink(曾经的toplink)，OpenJPA等可以使用，所以使用Jpa的一个好处是可以方便的切换ORM框架。

JPA与Hibernate的区别如下：

  • 可以理解为JPA是标准接口Hibernate是实现;
  • Hibernate实现与JPA的关系主要是通过三个组件来实现的hibernate-annotation、hibernate-entitymanager和hibernate-core:
    1. hibernate-annotation：是Hibernate支持annotation方式配置的基础，它包括了标准的JPA annotation以及Hibernate自身特殊功能的annotation。
    2. hibernate-core：是Hibernate的核心实现提供了Hibernate所有的核心功能。
    3. hibernate-entitymanager：实现了标准的JPA可以把它看成hibernate-core和JPA之间的适配器，它并不直接提供ORM的功能，而是对hibernate-core进行封装，使得Hibernate符合JPA的规范。

注意：也可以使用mvn dependency:tree命令可以查看。

在实现项目中直接引用org.hibernate:hibernate-entitymanager即可，从依赖关系包含hibernate-commons-annotations、hibernate-jpa和hibernate-core三个比较重要的依赖。

1. EntityManagerFactory

EntityManagerFactory是线程安全的对象，并且EntityManagerFactory的创建极其浪费资源，所以在使用JPA编程时，我们可以对EntityManagerFactory封装，只需要做到项目只存在一个EntityManagerFactory 即可。

实例化代码：

EntityManagerFactory factory = Persistence.createEntityManagerFactory("myJpa");

2. EntityManager

EntityManager是完成持久化操作的核心对象实体类作为普通java对象，只有在调用EntityManager将其持久化后才会变成持久化对象。

EntityManager对象在一组实体类与底层数据源之间进行对象关系映射的管理,它可以用来管理和更新Entity、查找Entity还可以通过JPQL语句查询实体。

我们可以通过调用EntityManager的方法完成获取事务以及持久化数据库的操作。

实例化代码：

EntityManager em = factory.createEntityManager();

3. EntityTransaction

EntityTransaction是完成事务操作的核心对象,在JPA中必须使用其相关方法实现数据持久化并让所有数据提交都支持事务在很大程度上保证了数据的一致性。

常用方法如下：

  1. begin：启动事务;
  2. commit：提交事务;
  3. rollback：撤消(回滚)事务;
  4. setRollbackOnly：使当前事务只能被撤消;
  5. getRollbackOnly：查看当前事务是否设置了只能撤消标志;
  6. isActive：查看当前事务是否是活动的。

实例化代码：

EntityTransaction tx = em.getTransaction();

4. 常用注解

  1. @Entity:表示将实体与表映射;
  2. @Table:表示数据库表名;
  3. @Id:表示数据库主键;
  4. @IdClass:表示联合主键;
    • 必须实现Serializable接口;
    • 必须有默认的public无参数的构造方法;
    • 必须覆盖equals和hashCode方法;
  5. @GeneratedValue:表示主键生成策略;
    • TABLE:JPA框架支持类型;
    • SEQUENCE:oracle支持类型;
    • IDENTITY:mysql支持类型;
    • AUTO:自动选择支持的类型;
  6. @Basic:表示属性是到数据库表的字段的映射;
  7. @Transient:表示该属性并非映射到数据库;
  8. @Column:表示数据表列名;
  9. @Tempral:表示数据库Date类型的精度;
    • TemporalType.DATE:映射为日期;
    • TemporalType.TIME:映射为时间;
    • TemporalType.TIMESTAMP:映射为日期和时间;
  10. @Enumerated:表示枚举类型字段映射;
  11. @Lob:表示数据库支持的大对象类型映射;
