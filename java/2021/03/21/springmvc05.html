Spring MVC 验证器

2021-03-21

在Web应用程序中客户端做的数据有效性验证都不是安全有效的，必须在服务端也对数据的有效性进行验证。

SpringMVC自身对数据在服务端的验证可以通过Validation进行实现，验证客户端提交到服务端的数据按照指定的规则进行数据有效性验证。

对于满足要求的数据SpringMVC会把它保存在错误对象中，这些错误信息通过SpringMVC提供的标签在前端JSP页面上进行展示。

在Spring框架诞生初期就已经有了数据验证功能，比JSR303的Java验证规范还要早，但是Spring的Validation功能在项目开发中比较常见，如果在新项目中还是建议使用JSR303进行验证。

1. Spring 验证器

自定义Spring验证器必须实现org.springframework.validation.Validator接口中的supports和validate方法，如果验证器指定的Class，supports方法将返回true，validate方法会验证目标对象，并将验证错误信息填入Errors对象。

接口的声明：

package org.springframework.validation;
public interface Validator {
    boolean supports(Class<?> clazz);
    void validate(Object target, Errors errors);
}

Errors对象是org.springframework.validation.Errors接口的一个实例，Errors对象中包含验证的FieldError和ObjectError对象，FieldError表示与被验证对象中的某个属性相关的一个错误。

在Errors对象上调用reject或者rejectValue方法添加错误信息。调用reject会给FieldError中添加一个ObjectError和rejectValue。

通过org.springframework.validation.ValidationUtils工具类可以简化开发者编写Spring验证器。

对于Errors对象中的错误消息可以使用表单标签库的Errors标签显示在HTML页面中。

方法签名：

ObjectError(String objectName, String[] codes, Object[] arguments,String defaultMessage);
FieldError(String objectName, String field, Object rejectedValue,boolean bindingFailure, String[] codes, Object[] arguments,String defaultMessage)

1.1. Spring的Validator实例

在Controller类中通过@InitBinder标记的方法只有在请求当前Controller的时候才会被执行，所以其中定义的Validator也只能在当前Controller中使用。

如果Validator对所有的Controller都起作用通过WebBindingInitializer的initBinder方法来设定了。

在SpringMVC的配置文件中通过mvc:annotation-driven的validator属性也可以指定全局的Validator。

实体类

public class Student {  
   
    private String name;  
     
    private String age;  
   
    public String getName() {  
       return name;  
    }  
   
    public void setName(String name) {  
       this.name = name;  
    }  
   
    public String getAge() {  
       return age;  
    }  
   
    public void setAge(String age) {  
       this.age = age;  
    }  
     
    public String toString() {  
       return name + ", " + age;  
    }  
     
}

验证类

import org.springframework.validation.Errors;  
import org.springframework.validation.ValidationUtils;  
import org.springframework.validation.Validator;  
   
public class PersonValidator implements Validator {  
   
    public boolean supports(Class<?> clazz) {  
       return Person.class.equals(clazz);  
    }  
   
    public void validate(Object obj, Errors errors) {  
       ValidationUtils.rejectIfEmpty(errors, "name", null, "name is empty.");  
       Person person = (Person) obj;  
       if (null == Person.getAge() || "".equals(Person.getAge()))  
           errors.rejectValue("age", null, "age is empty.");  
    }  
   
}

控制器

import javax.validation.Valid;  
import org.springframework.stereotype.Controller;  
import org.springframework.validation.BindingResult;  
import org.springframework.validation.DataBinder;  
import org.springframework.web.bind.annotation.InitBinder;  
import org.springframework.web.bind.annotation.RequestMapping;  
   
@Controller  
public class PersonController {  
     
    @InitBinder  
    public void initBinder(DataBinder binder) {  
       binder.setValidator(new PersonValidator());  
    }  
   
    @RequestMapping("list")  
    public String add(@Valid Person person, BindingResult result) {  
       if (result.hasErrors())  
           return "redirect:person/list";
       return "redirect:person/add";  
    }  
     
}

2. JSR303验证

JSR 303“Bean Validation”发布于2009年11月和JSR 349“Bean Validation 1.1”（发布于2013年5月）指定了一整套API通过注解给对象属性添加约束。

  • JSR 303下载地址：http://jcp.org/en/jsr/detail?id=303
  • JSR 349下载地址：http://jcp.org/en/jsr/detail?id=349

JSR只是一个规范文档本身无法使用必须编写实现类才能使用对于JSR bean validation的第一个实现是Hibernate Validator目前版本为5。

  • 下载地址：http://sourceforge.net/projects/hibernate/files/hibernate-validator/

第二个实现则是Apache BVal。

  • 下载地址：http://bval.apache.org/downloads.html

JSR 303不需要编写验证器但要利用JSR 303注解类型嵌入约束;约束属如下：

属性 描述 实例
@AssertFalse 应用于 boolean 属性，该属性值必须为 False @AssertFalse boolean hasChildren;
@AssertTrue 应用于 boolean 属性，该属性值必须为 True @AssertTrueboolean isEmpty;
@DecimalMax 该属性值必须为小于或等于指定值的小数 @DecimalMax(“1.1”)BigDecimal price;
@DecimalMin 该属性值必须为大于或等于指定值的小数 @DecimalMin(“0.04”)BigDecimal price;
@Digits 该属性值必须在指定范围内。integer 属性定义该数值的最大整数部分，fraction 属性定义该数值的最大小数部分 @Digits(integer=5,fraction=2) BigDecimal price;
@Future 该属性值必须是未来的一个日期 @Future Date shippingDate;
@Max 该属性值必须是一个小于或等于指定值的整数 @Max(150) int age;
@Min 该属性值必须是一个大于或等于指定值的整数 @Max(150)int age;
@NotNull 该属性值不能为 Null @NotNull String firstName;
@Null 该属性值必须为 Null @Null String testString;
@Past 该属性值必须是过去的一个日期 @PastDate birthDate;
@Pattern 该属性值必须与指定的常规表达式相匹配 @Pattern(regext=”\d{3}”) String areaCode;
@Size 该属性值必须在指定范围内 Size(min=2, max=140)String description;

2.1. JSR303实例

实体类

public class Student {  
   
    @NotNull(message="名字不能为空")
    private String name;  
     
    private String age;  
   
    public String getName() {  
       return name;  
    }  
   
    public void setName(String name) {  
       this.name = name;  
    }  
   
    public String getAge() {  
       return age;  
    }  
   
    public void setAge(String age) {  
       this.age = age;  
    }  
     
    public String toString() {  
       return name + ", " + age;  
    }  
     
}

控制器

import javax.validation.Valid;  
import org.springframework.stereotype.Controller;  
import org.springframework.validation.BindingResult;  
import org.springframework.validation.DataBinder;  
import org.springframework.web.bind.annotation.InitBinder;  
import org.springframework.web.bind.annotation.RequestMapping;  
   
@Controller  
public class PersonController {  
     
    @InitBinder  
    public void initBinder(DataBinder binder) {  
       binder.setValidator(new PersonValidator());  
    }  
   
    @RequestMapping("add")  
    public String add(@Valid Person person, BindingResult result) {

        if (result.hasErrors()){
            List<ObjectError> errorList = result.getAllErrors();
            for(ObjectError error : errorList){
                 System.out.println(error.getDefaultMessage());
            }
            return "redirect:person/list";  
        }
        return "redirect:person/add";  
    }  
     
}

注意：这里一个@Valid的参数后必须紧挨着一个BindingResult参数，否则spring会在校验不通过时直接抛出异常。

注意：启用注解驱动<mvc:annotation-driven />。
