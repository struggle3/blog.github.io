Spring MVC 注解类型

2021-03-18

Spring 2.5版本以后可以使用基于控制器注解类型方法进行业务开发，Controller和RequestMapping注释类型是Spring MVC API最重要的两个注解类型，RequestMapping注释可以对一个方法进行请求处理。

使用基于注解的控制器的如下优点：

  1. 控制器类可以处理多个动作，这就允许将相关的操作写在同一个控制器类中，从而减少应用程序中类的数量。
  2. 基于注解的控制器不需要存储在配置文件配置映射请求。

1. Controller 注解类型

使用@Controller注解修饰一个普通的Spring类为控制器。

@Controller
public class HomeController {}

1.1. 配置组件扫描

需要在Spring MVC的配置文件中将控制器注入到Sping IOC容器中，保证控制器能在项目中正常使用并处理相关请求。

在servlet-context.xml可以使用<context:component-scan base-package=”com.controller”/>和<bean class=”com.controller.HomeController”/>的两种方法进行配置。

2. RequestMapping 注解类型

RequestMapping是一个用来处理请求地址映射的注解可用于类或方法上。用于类上表示类中的所有响应请求的方法都是以该地址作为基地址。

package com.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.servlet.ModelAndView;

@Controller
public class HomeController {

    @RequestMapping(value = "home")
    public ModelAndView Home()
    {
        ModelAndView model=new ModelAndView();
        return  model;
    }
}

2.1. 参数说明

  1. value:请求的实际地址
  2. method:请求的method类型包含GET、POST、PUT、DELETE等
  3. consumes:请求的提交内容类型（Content-Type）
  4. produces:返回的内容类型，仅当request请求头中的(Accept)类型中包含该指定类型才返回
  5. params:指定request中必须包含某些参数值是，才让该方法处理
  6. headers:指定request中必须包含某些指定的header值，才能让该方法处理请求

2.2. 返回类型

  1. ModelAndView:跳转页面并传递数据
  2. Model:传递数据
  3. String:视图名称
  4. void:传递数据不跳转（ajax比较常用）

2.3. 请求参数中文乱码

如果在请示包含中文字符出现乱码现象则需要配置过滤器，而Spring框架本身也默认提供一个org.springframework.web.filter.CharacterEncodingFilter过滤器。

通常我们在web.xml中配置过滤器即可解决Spring MVC的请求参数的中文乱码问题。保证过滤器注册在其它过滤器之前，因为过滤器的执行是按照其注册顺序进行的。

<!--编码过滤-->
    <filter>
        <filter-name>encoding</filter-name>
        <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
        <init-param>
            <param-name>encoding</param-name>
            <param-value>UTF-8</param-value>
        </init-param>
        <init-param>
            <param-name>forceRequestEncoding</param-name>
            <param-value>true</param-value>
        </init-param>
        <init-param>
            <param-name>forceResponseEncoding</param-name>
            <param-value>true</param-value>
        </init-param>
    </filter>
    <filter-mapping>
        <filter-name>encoding</filter-name>
        <url-pattern>/*</url-pattern>
    </filter-mapping>

2.4. 校正请求参数名

在系统开发过程可能存在参数名称与处理方法中指定的参数名称不一致则需在处理方法参数前添加@RequestParam(“请求参数名”)注解指定请求URL所携带参数的名称。

在JSP页面中使用HTML表单input标签的name表示请求参数名。参见如下实例：

<form action="requestParam.do" method="post">
    姓名：<input type="text" name="userName"> <br/>
    年龄：<input type="text" name="userAge"> <br/>
    <input type="submit" value="提交参数">
</form>

package com.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.servlet.ModelAndView;

@Controller
public class HomeController {

    @RequestMapping(value = "/requestParam.do")
    public ModelAndView requestParam(@RequestParam(value = "userName",required = false) String name,
                                     @RequestParam(value = "userAge",required = false) Integer age){
        ModelAndView mv  = new ModelAndView();
        mv.addObject("name",name);
        mv.addObject("age",age);
        mv.setViewName("home");
        return mv;
    }
}

2.5. 应用@Autowired 和@Service 进行依赖注入

使用Spring 框架的一个好处是方便进行依赖注入Spring框架就是一个依赖注入容器，将依赖注入到Spring MVC控制器的最简单方法是添加注解@Autowired。

Autowired注解类型属于org.springframework.beans.factory.annotation包的成员,Autowired注解类型表示字段或方法上注入javabean。

Service注解该类型属于org.springframework.stereotype包的成员，Service注解类型表示类是一个服务。

package com.service;

import org.springframework.stereotype.Service;

@Service
public class StudentService {
}


package com.controller;

import com.bean.Student;
import com.service.StudentService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.ModelAttribute;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.servlet.ModelAndView;

@Controller
public class HomeController {

    @Autowired
    private StudentService studentService;

}
