Spring简介

2021-03-01

1. Spring框架是什么

Spring最初是由Rod Johnson编写的，并且于2003年6月首次在Apache 2.0许可下发布的一个开源的Java Web应用解决方案,目的是解决Web应用开发的复杂性而创建的。

Spring的核心是控制反转（IoC）和面向切面编程（AOP）。Spring是Java SE/EE开发平台下的轻量级开源框架。

Spring的主要作用就是为代码“解耦”降低代码间的耦合度。就是让对象和对象之间关系不是使用代码关联而是通过配置的形式进行表达。

Spring根据业务系统的功能特点使用Ioc降低业务对象之间耦合度。IoC使得主业务在相互调用过程中，不用再自己维护关系了，即不用再自己创建要使用的对象了。 而是由Spring容器统一管理自动“注入”（注入即赋值）而AOP使得系统级服务得到了最大复用，且不用再由程序员手工将系统级服务“混杂”到主业务逻辑中了而是由容器统一完成。

2. Spring优点？

Spring类似于一个Web开发脚手架由Spring Boot、Spring Cloud、Spring Security、Spring Data等许多模块组成。

2.1. 轻量

Spring框架使用的jar都比较小，一般在1M以下或者几百kb,Spring核心功能的所需的jar总共在3M左右。

Spring框架运行占用的资源少，运行效率高，不依赖其他jar。

2.2. 面向接口编程

Spring提供了Ioc控制反转由容器创建对象并管理对象之间的的依赖关系，将程序代码中的对象创建由容器完成实现对象之间的依赖解耦合。

2.3. AOP编程的支持

通过Spring提供的AOP功能，方便进行面向切面的编程，许多不容易用传统OOP实现的功能可以通过AOP轻松应付在Spring中，开发人员可以从繁杂的事务管理代码中解脱出来，通过声明式方式灵活地进行事务的管理，提高开发效率和质量。

2.4. 框架集成

Spring可以很方便的集成各种开源框架同时降低各种框架的使用难度并在一定程序上简化和统一了使用方法，Spring提供了对各种框架集成接口（如Struts,Hibernate、MyBatis）等的。
