spring boot发展路线

2021-03-01

1. Spring1.x

在Spring1.x时代都是通过xml文件配置bean，需要将xml配置分放到不同的配置文件中，在开发过程中需要在java和xml配置文件中频繁切换。

2. Spring2.x

JDK1.5带来的注解支持Spring2.x可以使用注解对Bean进行声明和注入减少了xml配置文件。

最佳实践：

  1. 基本配置用xml,比如：数据源、资源文件等；

  2. 业务开发用注解，比如：业务层（Service）中注入bean等

3. Spring3.x到Spring5.x

从Spring3.x开始提供了java配置方式，使用java配置方式可以更好的理解项目中所配置的Bean并且Spring4.x Spring5.x和Spring Boot都推荐使用java配置的方式。

4. Spring5.x零配置开发

Spring框架从Spring5.x版本推荐使用注解形式来对java应用程序进行开发与配置，并且可以完全替代原始的XML+注解形式的开发。

Spring框架提供了针对环境配置与业务bean开发相关注解

5. 注解

5.1. 声明Bean注解
  1. @Component:表示类是组件被Spring Ioc容器维护
  2. @Service:表示类是业务逻辑层(Service)
  3. @Repository:表示类是数据访问层(Dao)
  4. @Controller:表示类是展现层(MVC)
5.2. 注入Bean注解
  1. @Autowired: Spring官方提供注解
  2. @Injext:JSR-33提供注解(标准制定方)
  3. @Resource：JSR-250提供注解

以上三种注解在Set方法或属性上声明，开发中更习惯声明在属性上，代码简洁清晰。

5.3. Spring5.x核心注解

基于Spring5.x注解配置方式简化了xml配置，应用程序开发与xml环境配置均通过相应注解来实现。

  1. @Configuration:表示类是一个配置类相当于xml配置文件
  2. @ComponentScan:自动扫描指定包下标注有@Repository,@Service,@Controller
  3. @Bean:作用于方法上相当于xml文件中标签
  4. @Value:获取properties文件指定key value值
