Spring 基于注解注入

2021-03-05

对于依赖注入使用注解后可以不再需要在Spring配置文件中声明bean实例。

Spring中使用注解需要在Spring配置文件中添加component-scan标签表示配置组件扫描器，用于在指定包的扫描路径。

  • xml配置文件名称空间指定：

    xmlns:context="http://www.springframework.org/schema/context"

  • 添加配置代码

    <!--    1，可以通过逗号，分号，空格进行base-package进行指定
      2，可以指定base-package包路径在父级包目录
      3、可以指定在具体目录中
-->
<context:component-scan base-package="org.example"></context:component-scan>

1. Component

在类上使用注解@Component该注解的value属性用于指定该bean的id值。

如果不指定value参数值，默认使用类名的做为bean的id值，使用驼峰命名方式。

Spring还提供了3个创建对象的注解：

  • @Repository用于DAO实现类进行注解

  • @Service用于对Service实现类进行注解

  • @Controller用于对Controller实现类进行注解

service类代码

package org.example.service.impl;

import org.example.dao.NewsItemDao;
import org.example.service.NewsItemService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class NewsItemServiceImpl implements NewsItemService {

    @Override
    public void showTitle() {
        System.out.println("service方法执行。");
    }
}

junit代码

@Test
    public  void testShowTitle()
    {
        ApplicationContext ctx=new ClassPathXmlApplicationContext("applicationContext.xml");
        NewsItemServiceImpl news=ctx.getBean("newsItemServiceImpl",NewsItemServiceImpl.class);
        news.showTitle();
    }

2. Value

需要在属性上使用注解@Value该注解的value属性用于指定要注入的值。

使用该注解完成属性注入时类中无需setter如果属性有setter则也可将其加到setter上。

实体代码

package org.example.service.dto;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

@Component
public class NewsItem {
    @Value("mytitle")
    private  String Title;

    @Override
    public String toString() {
        return "NewsItem{" +
                "Title='" + Title + '\'' +
                '}';
    }
}

junit代码

@Test
public void  testNewsItem()
{
    ApplicationContext ctx=new ClassPathXmlApplicationContext("applicationContext.xml");
    NewsItem newsItem =ctx.getBean("newsItem",NewsItem.class);
    System.out.println(newsItem);
}

3. Autowired

需要在引用属性上使用注解@Autowired该注解默认使用按类型自动装配Bean方式。

使用该注解完成属性注入时类中无需setter如果属性有setter方法则也可将其加到setter方法上。

service类代码

package org.example.service.impl;

import org.example.dao.NewsItemDao;
import org.example.service.NewsItemService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;
import org.springframework.stereotype.Service;

@Component
public class NewsItemServiceImpl implements NewsItemService {

    @Autowired
    private NewsItemDao newsItemDao;

    @Override
    public void showTitle() {
        newsItemDao.showTitle();
        System.out.println("service方法执行。");
    }
}

4. Autowired与Qualifier

可以在引用的属性使用@Autowired与@Qualifier注解，用于指定自定义bean的id，在类中不需要setter方法，但是也可以将其用在setter方法上。

@Autowired还有一个属性required默认值为true，表示当匹配失败后会终止程序运行，若将其值设置为false，则匹配失败将被忽略未匹配的属性值为null。

package org.example.service.impl;

import org.example.dao.NewsItemDao;
import org.example.service.NewsItemService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;
import org.springframework.stereotype.Service;

@Component
public class NewsItemServiceImpl implements NewsItemService {

    @Autowired
    @Qualifier("newsitem")
    private NewsItemDao newsItemDao;

    @Override
    public void showTitle() {
        newsItemDao.showTitle();
        System.out.println("service方法执行。");
    }
}

5. Resource

Spring提供了对jdk中@Resource注解的支持，@Resource注解可以按名称匹配Bean，也可以按类型匹配Bean，可在属性上也可在setter方法上。

5.1. byType注入引用类型属性

@Resource注解若不带任何参数，采用默认按名称的方式注入，按名称不能注入bean，则会按照类型进行Bean的匹配注入。

package org.example.service.impl;

import org.example.dao.NewsItemDao;
import org.example.service.NewsItemService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;
import org.springframework.stereotype.Service;

import javax.annotation.Resource;

@Component
public class NewsItemServiceImpl implements NewsItemService {

    @Resource
    private NewsItemDao newsItemDao;

    @Override
    public void showTitle() {
        newsItemDao.showTitle();
        System.out.println("service方法执行。");
    }
}

5.2. byName注入引用类型属性

@Resource注解指定其name属性则name的值即为按照名称进行匹配的Bean的id。

package org.example.service.impl;

import org.example.dao.NewsItemDao;
import org.example.service.NewsItemService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;
import org.springframework.stereotype.Service;

import javax.annotation.Resource;

@Component
public class NewsItemServiceImpl implements NewsItemService {

    @Resource(name = "newsitem")
    private NewsItemDao newsItemDao;

    @Override
    public void showTitle() {
        newsItemDao.showTitle();
        System.out.println("service方法执行。");
    }
}

6. 注解与XML的对比

  1. 注解优点：
    • 方便
    • 直观
    • 高效（编码简单）。

硬编码在代码中具有侵入性，修改代码后必须编译程序。

  1. XML优点：
    • 配置和代码是分离的
    • 在xml中做修改无需编译代码重启服务器即可更新配置。

xml的缺点是：编写麻烦，效率低。
