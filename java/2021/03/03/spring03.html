Spring Bean实例化

2021-03-03

Bean实例化对象与new运算符效果一样，都是根据指定的类型创建一个实例，不同是Spring的Bean对象创建过程是由IoC容器完成。

基于XML的配置方进行Bean对象实例化，在标签中的class属性指定实例化对象的类型。

Bean实例化注意要点：

  1. 标签中的Id属性必须唯一

  2. 标签中的Name属性不必唯一

  3. 标签中的Class属性可以是实现类、可以是抽象类、不可以是接口

  4. 标签中的Id和Name属性使用驼峰命名

  5. 标签可以为Bean指定别名

1. 构造器实例化

默认使用无参的构造器进行bean实例化，通过XML的配置方式实例化代码如下：

 <bean id="newsItemServiceImpl" class="org.example.NewsItemServiceImpl"></bean>

2. 静态工厂方法实例化

静态工厂方法创建的bean使用class属性来指定包含static工厂方法的类，并使用名为factory-method的属性来指定工厂方法本身的名称。

package org.example;

public class StaticFactory {
    public  static  NewsItemService createNewsItemService()
    {
        System.out.println("静态工厂");
        return  new NewsItemServiceImpl();
    }
}


    <bean id="newsItemServiceImpl" class="org.example.StaticFactory" factory-method="createNewsItemService"></bean>


   @Test
    public  void testShowTitle()
    {
        ApplicationContext ctx=new ClassPathXmlApplicationContext("applicationContext.xml");
        NewsItemServiceImpl news=ctx.getBean("newsItemServiceImpl",NewsItemServiceImpl.class);
        news.showTitle();
    }

3. 实例工厂方法实例化

实例工厂通过实例化工厂本身之后，在去实例化bean对象并通过factory-bean指定工厂之后其次通过factory-method指定工厂内部的实例化方法进行对象创建。

package org.example;

public class InstanceFactory {
    public  NewsItemService createNewsItemService()
    {
        System.out.println("实例工厂");
        return new NewsItemServiceImpl();
    }
}

<bean id="instanceFactory" class="org.example.InstanceFactory"></bean>

<bean id="newsItemServiceImpl" factory-bean="instanceFactory" factory-method="createNewsItemService"></bean>

@Test
public  void testShowTitle()
{
    ApplicationContext ctx=new ClassPathXmlApplicationContext("applicationContext.xml");
    NewsItemServiceImpl news=ctx.getBean("newsItemServiceImpl",NewsItemServiceImpl.class);
    news.showTitle();
}
