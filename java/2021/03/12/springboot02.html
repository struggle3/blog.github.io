Spring Boot配置文件

2021-03-12

1. 介绍

Spring Boot的配置文件用于配置应用程序相关设置，以便让开发者在不同的运行环境中运行相同的应用程序（比如:基容器化的应用部署），包括properties,yaml,环境变量以用命令行。

1.1. 配置文件默认名称：

Spring Boot应用程序在运行时自动加载名称为application.properties和application.yaml的配置文件，并按照优先级从低到高的方式进行覆盖。

  • 配置文件位置：
  1. optional:file:classpath:/
  2. optional:file:classpath:/config/
  3. optional:file:file:./
  4. optional:file:file:./config/
  5. optional:file:file:./config/*/
  6. optional:file:classpath:custom-config/
  7. optional:file:file:./custom-config/
  • 配置文件加载顺序
  1. jar中的yml
  2. jar外的yml
  3. jar中的profile
  4. jar外的profile

注意：可以使用–spring.config.name指定配置文件名称，可以使用–spring.config.location指定配置文件加路径。

1.2. Profile文件加载

Spring Boot还支持通过命名约定的方式来加载特定的配置文件（例如：application-{profile}），特定的配置文件始终都会覆盖默认的配置文件。

例如：

  1. application-dev.yml
  2. application-test.yml
  3. application-prod.yml

注意：只有application.yml文件才是有效配置文件，如果要使用profile文件需要在文件加入spring.profiles.active=dev方可使文件生效。

1.3. 配置文件读取

  1. 使用SpringApplication.setDefaultProperties方式读取 
     package com.springboot01;

 import org.springframework.boot.SpringApplication;
 import org.springframework.boot.autoconfigure.SpringBootApplication;
 import org.springframework.context.annotation.PropertySource;
 import org.springframework.context.annotation.PropertySources;

 import java.io.IOException;
 import java.io.InputStream;
 import java.util.Properties;

 @SpringBootApplication
 public class Springboot01Application {

     public static void main(String[] args) throws IOException
     {
         SpringApplication springApplication=new SpringApplication(Springboot01Application.class);
         //创建Properties
         Properties properties=new Properties();
         //通过类读取器加载当前类下面的app.properties
         InputStream inputStream=Springboot01Application.class.getClassLoader()
                 .getResourceAsStream("app.properties");
         //加载输入中的properties
         properties.load(inputStream);

         springApplication.setDefaultProperties(properties);

         springApplication.run(args);

         //SpringApplication.run(Springboot01Application.class, args);
     }

 }
  2. 使用@PropertySource注解在@Configuration类方式读取 
    package com.springboot01;

 import org.springframework.boot.SpringApplication;
 import org.springframework.boot.autoconfigure.SpringBootApplication;
 import org.springframework.context.annotation.PropertySource;
 import org.springframework.context.annotation.PropertySources;

 import java.io.IOException;
 import java.io.InputStream;
 import java.util.Properties;

 @SpringBootApplication
 @PropertySource("classpath:app.properties")//通过注解加载配置
 public class Springboot01Application {

     public static void main(String[] args) throws IOException
     {
         SpringApplication springApplication=new SpringApplication(Springboot01Application.class);
         //创建Properties
         Properties properties=new Properties();
         //通过类读取器加载当前类下面的app.properties
         InputStream inputStream=Springboot01Application.class.getClassLoader()
                 .getResourceAsStream("app.properties");
         //加载输入中的properties
         properties.load(inputStream);

         springApplication.setDefaultProperties(properties);

         springApplication.run(args);

         //SpringApplication.run(Springboot01Application.class, args);
     }

 }

1.4 yaml说明

YAML 是 “YAML Ain’t Markup Language”（YAML 不是一种标记语言）的递归缩写。在开发的这种语言时，YAML的意思其实是：”Yet Another Markup Language”（仍是一种标记语言）。

1.4.1. 基本语法
  • key: value；kv之间有空格
  • 大小写敏感
  • 使用缩进表示层级关系
  • 缩进不允许使用tab，只允许空格
  • 缩进的空格数不重要，只要相同层级的元素左对齐即可
  • ’#’表示注释
  • 字符串无需加引号，如果要加，’‘与”“表示字符串内容 会被 转义/不转义
1.4.2. 数据类型
  1. 字面量：单个的、不可再分的值。date、boolean、string、number、null
k: v
  1. 对象：键值对的集合。map、hash、set、object
行内写法：  k: [v1,v2,v3]
#或者
k:
 - v1
 - v2
 - v3

</build>
