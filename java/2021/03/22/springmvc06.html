Spring MVC 表达式语言（EL）

2021-03-22

JSP最重要的特性之一就是表达式语言（EL）JSP 用户可以用它来访问应用程序数据。

由于受到ECMAScript和XPath表达式语言的启发，EL也设计成可以轻松地编写免脚本的JSP页面。

1. 语法

EL表达式以${开头，并以}结束，支持JavaBean、Map、List等对象操作，EL 表达式的结构如下： ${expression} #{expression}

  • 在JSF中${exp}会立即计算，#{expr}会延迟计算。

  • 在JSP中${exp}编译后页面执行时计算，#{expr}仅支持延迟表达标签中计算。

1.1. 关健字

关键字and、eq、gt、true、instanceof、or、ne、le、false、empty、not、lt、ge、null、div、mod不能用作标识符。

1.2. 运算符

EL表达式可以返回任意类型的值。如果EL表达式的结果是一个带有属性的对象，则可以利用”[]”或者”.”运算符来访问该属性。

1.3. 取值规则

EL表达式的取值是从左到右进行的对于expr-a[expr-b]形式的表达式其EL表达式的取值方法如下：

  1. 先计算expr-a得到value-a。
  2. 如果value-a为null则返回null。
  3. 然后计算expr-b得到value-b。
  4. 如果value-b为null则返回null。
  5. 如果value-a为java.util.Map则会查看value-b是否为Map中的一个key。若是则返回value-a.get(value-b)若不是则返回 null。
  6. 如果value-a为java.util.List或者array则要进行以下处理：
    • 强制value-b为int如果强制失败则抛出异常。
    • 如果value-a.get(value-b)抛出IndexOutOfBoundsException或者Array.get(value-a, value-b)抛出ArrayIndexOutOfBoundsException则返回null。
    • 否则若value-a是个List则返回value-a.get(value-b)若value-a是个array则返回Array.get(value-a, value-b)。
  7. 如果value-a不是一个Map、List或者array那么value-a必须是一个JavaBean。
    • 在这种情况下，必须强制 value-b 为 String。如果value-b是value-a的一个可读属性则要调用该属性的getter方法返回值。

2. EL 隐式对象

JSP页面中可以利用JSP脚本来访问JSP隐式对象，但是在没有脚本的JSP页面中，则不可能访问这些隐式对象。EL可以通过默认的隐式对象进行调用。

对象 描述
pageContext 当前JSP页面的javax.servlet.jsp.PageContext
initParam 所有环境初始化参数Map对象Key Value格式表示
param 所有请求参数Map对象Key Value格式表示
paramValues 所有请求参数并用参数Map对象Key Value格式，每个key的值是一个字符串数组，其中包含了指定参数名称的所有参数值
header 请求标题Map对象Key Value格式表示。每个key的值就是指定标题名称的第一个标题。获取多个标题用headerValues对象代替
headerValues 请求标题Map对象Key Value格式，每个key的值是一个字符串数组，其中包含了指定标题名称的所有参数值
cookie 当前请求对象中所有Cookie对象的Map。Cookie名称就是key名称每一个ke值就是一个Cookie对象
applicationScope ServletContext对象中所有属性的Map并用属性名称作为 key
sessionScope HttpSession对象中所有属性的Map，并用属性名称作为key
requestScope Map对象中包含了当前HttpServletRequest对象中的所有属性，并用属性名称作为 key
pageScope Map对象包含了全页面范围内的所有属性，属性名称就是Map的key

3. 其他EL运算符

除了”.”和”[]”运算符外EL还提供了其他运算符：算术运算符、关系运算符、逻辑运算符、条件运算符，以及empty运算符。

使用这些运算符时可以进行不同的运算，但是由于EL的目的是方便免脚本JSP页面的编程因此除了关系运算符外这些EL运算符的用处都很有限。

3.1. 算术运算符

  1. 加法（+）。
  2. 减法（−）。
  3. 乘法（*）。
  4. 除法（/和 div）。
  5. 取余/取模（%和 mod）。

除法和取余运算符有两种形式与XPath和ECMAScript是一致的。

注意:EL表达式的计算按优先级从高到低、从左到右进行。下列运算符是按优先级递减顺序排列的：

  • /div%mod
  • +-

这表示*、/、div、%以及 mod 运算符的优先级相同，+与−的优先级相同，但第二组运算符的优先级小于第一组运算符。

3.2. 关系运算符

  1. 等于（==和eq）
  2. 不等于（!=和ne）
  3. 大于（>和gt）
  4. 大于或等于（>=和ge）
  5. 小于（<和lt）
  6. 于或等于（<=和le）

3.3. 逻辑运算符

  1. 和（&&和and）
  2. 或（||和or）
  3. 非（!和not）

3.3. 条件运算符

EL 条件运算符的语法如下：

${先计算expr? A:B}

如果先计算expr的计算结果为True，那么该表达式的输出结果就是A否则为B。

3.3. empty运算符

empty运算符用来检查某一个值是否为null或者empty。

范例：

${empty X}

如果X为null或者X是一个长度为0的字符串，那么该表达式将返回True。如果X是一个空Map、空数组或者空集合它也将返回True否则将返回False。

3.3. 字符串连接运算符

+=运算符用于连接字符串。

范例：

${a+=b}
