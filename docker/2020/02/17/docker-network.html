docker网络管理

2020-02-17

1. 介绍

docker允许通过外部访问容器或容器互联的方式来提供网络服务，在一般使用中通过docker网桥与NAT的通信模板。

从图中可以看出在Host主机中会一个网口和docker0进行通信，docker0就类似于一个交换机用于交换不同容器之间的数据交互从而完成通信。

在docker0创建是会生成一个network namespace（命称空间）进行网络隔离一端在容器内部（eth0）另一端在Host主机中这样就是实现了容器与Host主机交互。

注意：docker0的ip地址就是容器的网关地址，容器的ip地址也是根据docker0进行分配的。

1.1. 外部网络访问容器(DNAT)

docker run -d -p 80:80 镜像

iptables -t nat -A PREROUTING -m addrtype --dst-type LOCAL -j DOCKER

iptables -t nat -A DOCKER ! -i docker0 -p tcp -m tcp --dport 80 -j DNAT --to-destination 172.17.0.2:80

1.2. 容器访问外部网络(SNAT)

iptables -t nat -A POSTROUTING -s 172.17.0.0/16 -o docker0 -j MASQUERADE

2. 网络参数修改

可以根据实际情况在必要的时候进行相关配置。

2.1. 进程网络修改

如果修改了进程的网络参数所有进程间的容器参数也会被修改。

-b, –bridge=”” 通过此参数可以自定义网桥设备，默认情况下docker会自动创建和使用docker0网桥设备。

–bip 指定docker0的IP和掩码使用标准的CIDR形式:如 11.11.11.11/24。

–dns 配置容器的DNS在启动docker进程是添加所有容器全部生效。

2.2. 容器网络修改

容器网络修改只对当前容器生效。

–dns 用于指定启动的容器的DNS

–net 用于指定容器的网络通讯方式有以下四个值

  • bridge：docker默认方式是网桥模式
  • none：容器没有网络栈
  • container：使用其它容器的网络栈docker容器会加入其它容器的network namespace
  • host：表示容器使用Host的网络没有自己独立的网络栈,容器可以完全访问Host的网络,安全性较低。

2.3 暴露端口

-p /P选项的使用格式

-p :ContainerPort将制定的容器端口映射至主机所有地址的一个动态端口

-p HostPort:ContainerPort映射至指定的主机端口

-p IP:HostPort:ContainerPort映射至指定的主机IP的主机端口

-p IP::ContainerPort映射至指定的主机的IP的动态端口

-P :暴露所需要的所有端口(大P)

备注:docker port ContainerName 可以查看容器当前的映射关系

2.4. 自定义docker0网桥地址

修改/etc/docker/daemon.json文件

{
"bip": "192.168.1.5/24",
"fixed-cidr": "10.20.0.0/16",
"fixed-cidr-v6": "2001:db8::/64", "mtu": "1500",
"default-gateway": "10.20.1.1",
"default-gateway-v6": "2001:db8:abcd::89", "dns": ["10.20.1.2","10.20.1.3"]
}

3. 网络隔离

常见的方式是通过网络名称空间进行隔离。

docker network ls 查看当前可用的网络类型

docker network create -d <类型> <网络空间名称>

类型：

docker network ls 查看当前可用的网络类型

docker network create -d 类型 网络空间名称

3.1. 创建两个网桥

docker network create -d bridge lamp
docker network create -d bridge lanp

3.2. 查看网络

docker network ls

3.3. 运行镜像

docker run -p 81:80  --name nginx1 -d --network=lamp nginx
docker run -p 82:80  --name nginx2 -d --network=lanp nginx

4. Linux桥接器进行主机间通讯

4.1. 通过配置文件修改

1.复制ifcfg-ens33

cp -a etc/sysconfig/network-scripts/ifcfg-ens33 ifcfg-br0

2.编辑ens33如下

删除IP相关的配置并添加一个BRIDGE=br0即可。

TYPE=Ethernet
PROXY_METHOD=none
BROWSER_ONLY=no
BOOTPROTO=static
DEFROUTE=yes
IPV4_FAILURE_FATAL=no
IPV6INIT=yes
IPV6_AUTOCONF=yes
IPV6_DEFROUTE=yes
IPV6_FAILURE_FATAL=no
IPV6_ADDR_GEN_MODE=stable-privacy
NAME=ens33
UUID=3911e1b4-77c2-46bf-8e1d-c6f930688a70
DEVICE=ens33
ONBOOT=yes
BRIDGE=br0

3.编辑br0

DEVICE=br0
TYPE=Bridge  
ONBOOT=yes 
BOOTPROTO=static 
IPADDR=192.168.80.139 
NETMASK=255.255.255.0 
DNS=8.8.8.8

4.重启网络

service network restart

这样ens33就没有地址br0就是一个可以正式工作的地址,但是docker并不能借助这个地址去通信。

4.2. pipework

Jérôme Petazzoni 编写了一个叫 pipework 的 shell 脚本，可以帮助用户在比较复杂的场景中完成容器的连接。

yum install -y git
git clone https://github.com/jpetazzo/pipework [root@localhost network-scripts]
cp pipework/pipework /usr/local/bin/
chmod a+x /usr/local/bin
docker run --name nginx1  --net=none -d nginx #--net=none设置为空的目是为了让pip去指定
pipework br0 nginx1 192.168.80.135/24 #这个命令是不是永久生效的,需要加在开机脚里面
