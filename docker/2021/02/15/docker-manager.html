docker容器管理

2021-02-15

基本概念

Docker 包括三个基本概念:镜像（Image）、容器（Container）、仓库（Repository）理解了这三个概念，就理解了 Docker 的整个生命周期。

  1. Docker 镜像（Image）是模板类似于生活中楼房的图纸建筑工人根据图纸建造楼房同时也和程序中类（class）相似，程序员可以通过类（class）创建相关实例。
  2. Docker 容器（Container）是镜像创建并运行的实例，可以当作为一个迷你的Linux系统和运行的相关应用程序。
  3. Docker 仓库（Repository）是存放镜像文件的地方，类似于程序中的对象创建工厂和IOC容器（可能比喻不是特别恰当）及git。

基础命令

命令是运维人员与docker交互的入口,命令除了单条使用外，还支持赋值、解析变量、嵌套使用。

序号 名称 作用
1 docker info 守护进程的系统资源设置
2 docker search 仓库的查询
3 docker pull 仓库的下载
4 docker images 镜像的查询
5 docker rmi 镜像的删除
6 docker ps 容器的查询
7 docker run 容器的查询
8 docker start 容器启动
9 docker stop 容器停止

单个容器管理

每个容器被创建后都会分配一个ID作为容器的唯一标示后续对容器的启动、停止、修改、删除等所有操作都是通过ID来完成类似数据库概念中的主键。

序号 名称 作用
1 docker ps –no-trunc 查看
2 docker stop/start CONTAINERID 停止
3 docker start/stop webapp 通过容器别名启动/停止
4 docker inspect webapp 通过容器别名启动/停止
5 docker logs webapp 查看容器日志
6 docker stats webapp 查看容器所占用的系统资源
7 docker exec 容器名 容器内执行的命令 容器执行命令
8 docker exec -it 容器名 /bin/bash 登入容器的bash
9 –restart=always 容器的自动启动
10 -h x.xx.xx 设置容器主机名
11 –dns xx.xx.xx.xx 设置容器使用的 DNS 服务器
12 –dns-search DNS 搜索设置
13 –add-host hostname:IP 注入 hostname <> IP 解析
14 –rm 服务停止时自动删除
